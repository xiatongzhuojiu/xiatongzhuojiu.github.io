ls -al 显示当前目录下的所有的文件及属性

mkdir 创建文件夹

mkdir -p 文件名/文件名/文件名创建递归文件夹

cd切换目录

whoami查看当前用户的名字

cat 文件名 查看文本文件

id当前用户的权限

pwd当前所在的目录

rm -f 文件名 强制删除

rm -rf test1/ 强制删除文件夹下所有的内容

ls /bin |grep mv查找bin目录所有带mv的文件

ls /bin|more

ls /bin|less

ls /bin|grep ls

cat /bin/ls

touch anaconda-ks.cfg

cat -n 文件名 查看文件内容（-n显示行号）

chmod 755 文件名 修改权限

find /bin -name ls搜索在bin目录下名字为ls的文件

find /bin -user root

find /bin size +204800 搜索bin目录中大小在100M以上的文件

find /bin -iname ls

useradd admin 创建用户

二. picgo和typora的连接

  1. 打开typora,选择文件->偏好设置

  1. 点击图像,插入图片时选择第三个,然后勾选前四个,上传服务选择PicGo(app),PicGo路径精确定位到PicGo.exe

  1. 在github上点击个人主页,选择setting
  2. 找到develoer setting->Personal access tokens->Tokens(classic)->Generate new token->Generate new token(classic),命名为123,生效时间选择永久,勾选第一个,生产令牌后复制令牌

  1. 打开PicGo,点击图床设置->GitHub,然后编辑

  1. 图床配置名为123,仓库名为github用户名/仓库名,分支名设为main,Token处填写刚刚复制的令牌,设定存储路径为img/,点击确定,然后设为默认图床

  1. 回到typora,点击验证图片上传选项,会显示验证成功,这样PicGo和typora的连接就建立成功了.
